# Sleepy History Codex Harness Plan

Created: 2026-05-09

Purpose: Build a native iOS bedtime history app that generates original long-form narrated historical sleep stories, keeps provider integrations modular, and can be deployed to a real iPhone with minimal operational friction.

## Research Notes

- Inspiration: Boring History For Sleep by HistoryAndSleepOfficial uses very long calm historical sleep episodes, gentle narration, ambience such as rain or waves, chaptered topics, historical figures, daily life, and "quiet" factual storytelling. We should be inspired by the product pattern, not copy scripts, branding, voice identity, episode titles, or exact phrasing.
- Initial provider intent: Gemini 3.1 Pro for research dossiers, Claude Opus 4.6 for story writing, ElevenLabs for narration, GPT-Image 2 for cover art.
- Provider caveat: provider names drift. The build must use a provider registry with configurable model IDs, runtime capability checks, and fixture-based mocks. Opus 4.6 should default to `claude-opus-4-6`. GPT-Image 2 should default to `gpt-image-2`. Gemini research should verify the exact available Gemini 3.1 Pro model ID through the provider API before implementation, with a configured fallback only after user approval.
- Recommended architecture: native SwiftUI iOS client plus a small TypeScript backend/worker. The backend keeps API keys off-device, handles long-running story jobs, and exposes simple status/download APIs. Start with a single-worker queue and persisted checkpoints before adding Redis or heavier infrastructure.
- Initial product stance: personal-first, local library, original generated content, no public social sharing, no creator marketplace, no account system unless needed for deployment or sync.
- Target device: iPhone 14 Pro Max running iOS 26. Use iOS 26 as the MVP deployment target unless broader compatibility becomes a goal.
- Hosted generation must be protected before any real provider deployment: per-device owner auth, allowlisting, job limits, provider budget caps, retry ceilings, audit logs, and an emergency kill switch.
- Source verification should prefer Gemini grounding metadata. Any backend URL fetch must go through a small safe source fetcher that only stores sanitized citation metadata.

## Open Decisions

- Recommended first install: direct Xcode install with automatic signing for the iPhone 14 Pro Max; revisit TestFlight once the MVP is stable enough for regular beta builds.
- Choose the default narration voice and whether ambient rain/fire/ocean audio should ship in MVP or remain a later toggle.
